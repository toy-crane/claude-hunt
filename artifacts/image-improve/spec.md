## Overview
프로젝트 보드 데스크톱 리스트의 작은 썸네일은 흰 배경 스크린샷이 많아 식별이 어렵다. 행을 호버하면 같은 스크린샷의 큰 미리보기가 옆에 떠올라, 정보 밀도를 유지한 채 식별성을 보강한다.

## Scope

### Included
- 데스크톱(≥720px) 프로젝트 보드 리스트의 썸네일에 호버 미리보기 추가
- 미리보기는 동일 스크린샷의 확대 이미지로 행 우측에 떠오름
- 마우스 호버와 키보드 포커스 모두 트리거로 동작
- 등장과 사라짐에 부드러운 트랜지션 적용

### Excluded
- 모바일(<720px) 적용 — 터치 환경에는 호버 개념이 없고, 이미 풀폭 썸네일이라 식별 문제 없음
- 썸네일 자체의 크기/스타일 변경 — 리스트 정보 밀도를 유지하기 위해 작은 썸네일을 그대로 둠
- `prefers-reduced-motion` 별도 처리 — 첫 라운드에서는 단일 트랜지션 정의로 운영, 사용자 피드백 후 재검토
- 미리보기 안의 부가 메타(제목/태그라인/작성자) 표시 — 행에 이미 표시되므로 중복

## Scenarios

### 1. 호버 시 큰 미리보기 등장
**Given** 데스크톱(≥720px)에서 프로젝트 보드 리스트가 표시되고, 첫 번째 행이 스크린샷을 가지고 있다
**When** 사용자가 첫 번째 행의 썸네일에 마우스를 올린 채 약 150ms 머문다
**Then** 썸네일 우측에 같은 스크린샷의 큰 미리보기가 부드럽게 떠오른다

Success Criteria:
- [ ] 썸네일에 마우스를 올리고 200ms 후 미리보기 요소가 화면에 보인다
- [ ] 미리보기에 표시되는 이미지의 `src`는 썸네일 이미지의 `src`와 동일하다
- [ ] 미리보기는 행의 다른 컬럼(이름·작성자·투표 버튼)을 가리지 않는 위치에 표시된다
- [ ] 미리보기는 등장 시 페이드/줌이 100ms 이상에 걸쳐 보이며, 즉시 깜빡 나타나지 않는다

### 2. 마우스가 빠질 때 미리보기 닫힘
**Given** 미리보기가 열린 상태
**When** 사용자가 썸네일과 미리보기 영역에서 마우스를 완전히 빼낸다
**Then** 잠시 머문 뒤 미리보기가 부드럽게 사라진다

Success Criteria:
- [ ] 마우스가 영역을 벗어난 뒤 약 150ms까지는 미리보기가 유지된다(우발적 leave 보호)
- [ ] 그 이후에는 미리보기 요소가 DOM에서 제거되거나 보이지 않게 된다
- [ ] 사용자가 썸네일 → 미리보기 영역으로 마우스를 옮겨도 미리보기는 닫히지 않는다

### 3. 스크린샷이 없는 행에서는 미리보기 미표시
**Given** 데스크톱에서 프로젝트 보드 리스트가 표시되고, 한 행은 스크린샷이 없는 빈 플레이스홀더 썸네일을 가진다
**When** 사용자가 그 빈 썸네일에 마우스를 올린다
**Then** 미리보기가 뜨지 않는다

Success Criteria:
- [ ] 스크린샷 없는 행을 호버해도 미리보기 요소가 화면에 나타나지 않는다
- [ ] 같은 썸네일을 클릭했을 때의 기존 동작(프로젝트 URL 새 탭 이동)은 영향이 없다

### 4. 키보드 포커스로도 미리보기가 열림
**Given** 데스크톱에서 프로젝트 보드 리스트가 표시된다
**When** 사용자가 키보드 Tab으로 한 행의 썸네일 링크에 포커스를 이동한다
**Then** 마우스 호버와 동일하게 미리보기가 떠오른다

Success Criteria:
- [ ] 썸네일 링크에 키보드 포커스가 있는 동안 미리보기 요소가 보인다
- [ ] 포커스가 다른 요소로 이동하면 미리보기가 사라진다

### 5. 클릭 시 프로젝트 이동은 그대로
**Given** 미리보기가 열려 있든 닫혀 있든
**When** 사용자가 썸네일을 클릭한다
**Then** 새 탭으로 프로젝트 URL이 열린다(기존 동작 유지)

Success Criteria:
- [ ] 썸네일 클릭 시 `target="_blank"`로 프로젝트 URL이 열린다
- [ ] 미리보기가 열려 있는 상태에서 클릭해도 페이지 내비게이션이 차단되지 않는다

## Invariants
- **모바일(<720px) 영향 없음**: 모바일 카드의 풀폭 썸네일·레이아웃·인터랙션은 변경되지 않는다.
- **클릭 내비게이션 우선**: 미리보기 표시 상태(열림/닫힘/애니메이션 중)와 무관하게 썸네일 클릭은 항상 프로젝트 URL을 새 탭으로 연다.

## Dependencies
- 프로젝트 보드 리스트(데스크톱 그리드 행 + 썸네일)
- 프로젝트 스크린샷 업로드 기능(스크린샷 없는 행이 존재할 수 있음)
