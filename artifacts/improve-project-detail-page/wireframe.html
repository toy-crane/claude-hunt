로그인하고 의견 남기기
[😊+] 트리거 클릭 시 — 4개 이모지 팝오버
위치: 트리거 위/아래 (자동 flip)
댓글 0개일 때
아직 댓글이 없어요. 첫 의견을 남겨주세요
Notes

상세 페이지 하단의 댓글 섹션. 시나리오 3·4를 모두 커버. 스크롤하면 본문 화면 아래에 이어짐.

  • 레이아웃:
    • 섹션 헤더: "댓글 N" + 정렬 표시(최신순). 정렬은 v1엔 토글만 시각화, 변경 옵션은 미정
    • 입력 영역(인증 시): 아바타 + textarea + 카운터 + [취소][등록]. 비로그인 시엔 textarea 대신 "로그인하고 의견 남기기" 안내
    • 댓글 카드: 아바타 / [이름 · 시간 (· 수정됨)] / 본문 / 반응 row
    • 반응 row 패턴 (Slack/Reddit 스타일): count ≥ 1인 이모지 chip만 노출 + 점선 테두리의 [😊+] 트리거. 반응 0개일 땐 [😊+ 반응] 한 개만 보임. 트리거 클릭 → 4개(👍 💡 🎉 ❤️) 이모지 팝오버 → 선택 시 chip이 row에 추가/토글됨
    • "답글하기"는 반응 row의 *우측 정렬 텍스트 링크* — 좌측 반응 그룹과 시각적으로 분리
    • 답글: 부모 댓글 본문 안에 좌측 보더 + 들여쓰기로 1뎁스 노출. 답글 자체에는 "답글하기" 없음 (depth = 1)
    • 본인 댓글에만 우상단 ⋯(더보기) 메뉴 → 편집 / 삭제
    • "수정됨" 인디케이터: 시간 옆에 작은 italic 라벨
  • 인터랙션:
    • 이모지 chip 클릭: 토글 (이미 누른 거면 취소). count = 0이면 라벨만 보임, 1+이면 숫자 표시. 본인이 누른 chip은 검정 배경(.own)
    • "답글하기" 클릭: 부모 댓글 아래로 답글 입력 영역 펼쳐짐 (toplevel 입력과 동일 형태, 더 작게)
    • ⋯ 클릭: 작은 메뉴 — "편집" / "삭제(확인)". 다른 사용자 댓글에는 ⋯ 자체가 안 보임
    • 편집 클릭: textarea로 인라인 변환 → 저장하면 "수정됨" 라벨 추가
    • 삭제 클릭: 확인 다이얼로그 → 삭제 시 답글·반응까지 함께 사라짐 (스펙 invariant)
    • 비로그인 사용자: 댓글·답글·반응 모두 *읽기*만, chip 클릭은 무반응(또는 로그인 안내)
    • 1000자 초과: 등록 버튼 비활성 + 카운터 색 변화 + 인라인 에러 노출
  • 고려 안 한 항목 (스펙의 Excluded와 일치):
    • 알림: 새 댓글이 달려도 작성자에게 알림 없음 (v1)
    • 멘션 자동변환: @Bob을 작성자가 직접 타이핑하면 단순 텍스트로 저장
    • 신고/모더레이션: 없음
