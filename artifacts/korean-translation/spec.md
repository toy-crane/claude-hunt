# Korean Translation

## Overview

claude-hunt 사이트의 모든 영문 UI를 한국어로 번역한다. 수강생이 전원 한국인이므로 언어 전환 기능 없이 한국어 단일 언어로 운영하며, 동시에 추천(투표) 버튼을 텍스트 없는 아이콘으로 재구성한다. 자세한 표기·번역 원칙은 `notes.md` 참고.

## Scope

### Included
- 모든 페이지의 UI 문구(헤딩, 본문, 버튼, 라벨, placeholder, 빈 상태, 토스트, 검증 에러, 다이얼로그)를 한국어로 표시
- 페이지 메타데이터(`<title>`, description, OG 태그)를 한국어로 표시
- 접근성 속성(`aria-label`, alt text, 스크린리더 전용 텍스트)을 한국어로 표시
- 추천 버튼을 아이콘 단독(↑ 모양 + 긍정 컬러)으로 재구성하고 "Sign in to vote" 등 부속 텍스트를 모두 제거
- "Cohort" → "클래스"로 통일
- 외래어는 한글 표기로 통일 ("Project" → 프로젝트, "Profile" → 프로필, "Project Board" → 프로젝트 보드)

### Excluded
- 언어 전환(i18n) 기능 — 사용자가 전원 한국인이므로 불필요
- 사용자 생성 콘텐츠(프로젝트 제목/설명/태그라인, 프로필 표시명/bio) 번역 — 입력 시 이미 한국어이며 자동 번역할 이유 없음
- Supabase Auth 메일/OTP 본문 한국어화 — 외부 서비스 기본값 사용. 별도 작업으로 분리
- 클래스 명칭 외 정렬·필터 등 동작 변경 — 표시 텍스트만 다룸

## Scenarios

### 1. 처음 방문한 수강생이 한국어로 사이트를 본다
**Given** 비로그인 방문자가 사이트 루트에 처음 진입한다
**When** 홈 페이지가 렌더링된다
**Then** 헤딩·서브헤딩·헤더·푸터가 모두 한국어로 표시된다

Success Criteria:
- [ ] 홈 페이지 헤딩에 "프로젝트 보드"가 표시된다
- [ ] 홈 페이지 서브헤딩에 "좋아하는 프로젝트에 응원을 보내주세요." 가 표시된다
- [ ] 헤더의 비로그인 진입 버튼 라벨이 "로그인"이다
- [ ] 페이지에 "Project Board", "Cohort", "Discover", "Submit a project" 등 영문 UI 문구가 한 건도 노출되지 않는다 (브라우저 텍스트 검색 기준)

### 2. 수강생이 한국어 폼으로 프로젝트를 제출한다
**Given** 로그인된 수강생이 홈에서 제출 다이얼로그를 연다
**When** 다이얼로그 안의 모든 라벨·placeholder·버튼을 확인하고 빈 값으로 제출 시도한다
**Then** 라벨·검증 에러·토스트가 모두 한국어로 표시된다

Success Criteria:
- [ ] 다이얼로그 제목이 "프로젝트 제출"이다
- [ ] 필드 라벨이 "제목", "태그라인", "프로젝트 URL", "스크린샷"이다
- [ ] 빈 제목으로 제출 → 제목 필드 아래에 "제목을 입력해 주세요." (또는 동일 의미의 한국어 메시지) 가 표시된다
- [ ] `http://` 또는 `https://` 로 시작하지 않는 URL 입력 → "http:// 또는 https:// 로 시작하는 URL을 입력해 주세요." 가 표시된다
- [ ] 정상 제출 후 토스트 메시지가 한국어로 표시된다 (예: "프로젝트가 제출되었어요.")
- [ ] 모든 메시지가 친근한 존댓말(해요체)이며, "~하십시오"와 같은 합쇼체가 사용되지 않는다

### 3. 신규 수강생이 한국어 온보딩에서 클래스를 선택한다
**Given** 로그인 직후 온보딩 페이지로 이동한 수강생
**When** 폼의 모든 라벨과 dropdown 옵션을 확인한다
**Then** "Cohort"가 모두 "클래스"로 표시되고, 검증 에러와 안내 메시지도 한국어로 표시된다

Success Criteria:
- [ ] 페이지 헤딩이 "프로필 설정" (또는 동일 의미의 한국어) 이다
- [ ] 클래스 선택 필드 라벨이 "클래스"이다
- [ ] 클래스 dropdown placeholder가 "클래스를 선택하세요" 형태의 한국어 문구이다
- [ ] 표시명을 비운 채 제출 → "표시명을 입력해 주세요." 형태의 한국어 에러가 표시된다
- [ ] 클래스가 하나도 없을 때 안내 alert 본문이 한국어로 표시된다 ("클래스가 아직 없어요. 강사에게 클래스 배정을 요청해 주세요." 형태)
- [ ] 페이지에서 "Cohort" 단어가 어디에도 노출되지 않는다

### 4. 수강생이 아이콘만 있는 추천 버튼으로 프로젝트를 추천한다
**Given** 로그인된 수강생이 홈에서 프로젝트 카드를 본다
**When** 추천 버튼의 시각적 형태와 클릭 후 상태를 확인한다
**Then** 버튼은 텍스트 없이 위(↑)를 가리키는 아이콘과 긍정 컬러로 표시되고, 클릭 시 카운트가 즉시 증가한다

Success Criteria:
- [ ] 추천 버튼에 "Upvote", "추천", "투표", "Vote" 등 어떤 텍스트 라벨도 시각적으로 노출되지 않는다 (카운트 숫자는 별개)
- [ ] 추천 버튼 아이콘이 위 방향(↑)을 명확히 가리킨다
- [ ] 활성/포커스 상태에서 긍정 컬러(녹색·주황 등 따뜻한 계열)가 적용된다
- [ ] 버튼의 `aria-label`이 "추천하기" (또는 동일 의미의 한국어) 이다 — 스크린리더 사용자도 행동 의미를 들을 수 있다
- [ ] 클릭 후 카운트가 1 증가한다

### 5. 비로그인 방문자가 추천을 시도하면 로그인으로 유도된다
**Given** 비로그인 방문자가 홈에서 프로젝트 카드를 본다
**When** 추천 아이콘을 클릭한다
**Then** "Sign in to vote" 같은 부속 텍스트 없이 곧바로 로그인 플로우로 이동한다

Success Criteria:
- [ ] 비로그인 상태에서 추천 버튼 옆/안에 "Sign in to vote" 또는 동등한 영문 안내가 노출되지 않는다
- [ ] 비로그인 상태에서 추천 아이콘 클릭 → 로그인 페이지로 이동한다
- [ ] 로그인 페이지의 모든 라벨·버튼·구분선 텍스트가 한국어로 표시된다 (단 "GitHub", "Google" 버튼 라벨은 영어 유지)

### 6. 사이트 링크를 공유하면 한국어 미리보기가 노출된다
**Given** 외부 사용자가 메신저나 SNS에 사이트 URL을 공유한다
**When** 해당 플랫폼이 OG 태그를 읽어 미리보기를 렌더링한다
**Then** 제목과 설명이 한국어로 표시된다

Success Criteria:
- [ ] 페이지 `<title>` 이 한국어로 노출된다 (예: "프로젝트 보드 · claude-hunt", 단 "claude-hunt" 브랜드명은 영어 유지)
- [ ] 페이지 description 메타에 "함께 배우는 사람들의 프로젝트"가 노출된다
- [ ] OG title/description 메타가 한국어이다
- [ ] description 어디에도 "Discover", "cohort", "claude-hunt" 외 영문 카피가 포함되지 않는다

### 7. 계정 삭제 다이얼로그가 한국어로 위험을 알린다
**Given** 로그인된 수강생이 설정 페이지에서 "위험 영역"의 계정 삭제를 시도한다
**When** 삭제 확인 다이얼로그가 열린다
**Then** 삭제될 항목 목록과 확인 입력 안내가 모두 한국어로 표시된다

Success Criteria:
- [ ] 다이얼로그 제목이 "계정 삭제" (또는 동일 의미) 이다
- [ ] 삭제될 항목 4개(프로필, 제출한 프로젝트, 추천 기록, 업로드한 스크린샷)가 한국어 bullet으로 표시된다
- [ ] 확인 프롬프트가 "확인을 위해 [이메일]을 입력해 주세요." 형태의 한국어이다
- [ ] 취소/확정 버튼 라벨이 한국어이다 ("취소", "삭제")

## Invariants

- **브랜드/고유명사 보존**: "Claude Hunt", "Claude Code", "GitHub", "Google", "Built by toycrane", "claude-hunt" 도메인 표기는 어떤 화면에서도 영어 그대로 노출된다
- **사용자 생성 콘텐츠 무변경**: 사용자가 입력한 프로젝트 제목·태그라인·설명·표시명·bio는 번역되지 않고 입력값 그대로 표시된다
- **단일 언어 운영**: 사이트 어디에도 언어 전환 토글·드롭다운·라우트(`/en`, `/ko`)가 노출되지 않는다
- **외래어 표기 일관성**: "프로젝트", "프로필", "이메일", "라이트/다크/시스템" 등 외래어 한글 표기는 모든 화면에서 동일 표기를 사용한다 (한 화면은 "프로젝트", 다른 화면은 "Project" 식으로 섞이지 않는다)
- **톤 일관성**: 모든 UI 문구는 친근한 존댓말(해요체)을 사용한다. "~하십시오", "~합니다" 같은 합쇼체나 반말은 사용하지 않는다

## Dependencies

- 기존 인증, 프로젝트 제출, 투표, 온보딩, 설정 기능이 영어 UI로 이미 동작하고 있어야 한다 (이번 작업은 표시 텍스트와 추천 버튼 외형만 변경)

## Design Decisions (from wireframe)

- **추천 버튼 형태**: 옵션 C — 미니멀 outline. 현재 shadcn `outline` variant 유지, 텍스트만 제거하고 ThumbsUp 아이콘을 위(↑) 방향 아이콘으로 교체. 가로 정렬로 아이콘 + 카운트 표시. 활성 시 텍스트·테두리에만 긍정 컬러 적용 (배경은 그대로).

## Undecided Items

- 추천 버튼 긍정 컬러의 정확한 톤 (녹색 계열 vs 주황 계열) — 구현 단계에서 결정
- 토스트·검증 에러의 정확한 한국어 문안 — 일관된 어휘집(예: "표시", "선택", "입력") 기준은 구현 단계에서 확정
