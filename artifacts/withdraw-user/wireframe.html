Notes

Redesigned Settings page: three labeled cards in a single column.

  • Trigger: signed-in user navigates to /settings. Unauthenticated visitor is redirected to /login?next=/settings before this screen is reached.
  • Interaction: edit Display name → Save; click Log out in the Account card; click Withdraw in the Danger Zone to open the Withdraw dialog.
  • Transition: "Back to home" returns to "/". "Withdraw" opens the Withdraw · empty screen (modal over this page). "Log out" ends the session and returns to "/".
  • Section order and row order: Profile (Email → Display name → Save) · Account (Log out) · Danger Zone (Delete account).

Notes

Withdraw confirmation dialog, initial state. Confirm button is disabled.

  • Trigger: user clicks Withdraw on the Settings screen.
  • Interaction: type email to enable the Delete account button; click Cancel or dismiss to close without deleting.
  • Transition: Cancel/close → back to Settings (no changes). Exact email match → Withdraw · confirmed screen.
  • Also covers scenario-3 (cancel) since the initial state and the cancel path share this layout.

Notes

Withdraw dialog after the user has typed their exact email. Confirm button is enabled.

  • Trigger: typing an exact match of the viewer's email into the confirmation input.
  • Interaction: click Delete account to complete withdrawal; click Cancel to abort with no deletion.
  • Transition: Delete account → session ends, redirect to "/" (home). Cancel → back to Settings screen.
  • Case-sensitive match: "ALICE@EXAMPLE.COM" would not reach this state (still disabled).
